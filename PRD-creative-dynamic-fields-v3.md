# 创意部反馈驱动的动态字段能力优化 PRD

更新日期：2026-05-14

## 1. 背景

当前系统的定位不是为单一部门制作独立工作台，而是公司级运营需求中台。创意部反馈说明：不同部门、不同岗位确实会有差异化表单和处理方式，但这些差异不应该脱离主系统，而应该沉淀到动态字段、字段模板、需求类型、权限和视图能力里。

因此，本轮优化目标不是新增“创意部独立系统”，而是以创意部为第一个高复杂度场景，升级动态字段能力，使系统更接近可配置 SaaS 表单引擎。

## 2. 产品原则

1. 底层仍统一使用现有需求系统：`demands`、`departments`、`demand_types`、`department_field_templates`、`department_fields`。
2. 字段类需求优先沉到动态字段：必填、字段类型、字段配置、字段模板、字段组校验。
3. 交互类能力沉到字段组件：图片上传、素材库选择、文案模板、链接或上传二选一。
4. 视图类能力沉到需求列表和详情体验：岗位默认视图、上一条/下一条、返回列表状态保持。
5. 素材库/NAS 是公共素材能力，不做成创意部私有模块。

## 3. 第一阶段已优先处理

### 3.1 创意部岗位视图分流

目标：

- 设计师只看 UI / 美工 / Banner 类需求。
- 视频剪辑师只看视频剪辑需求。
- 创意部负责人、管理员仍可看全部创意需求。

当前实现：

- 后端 `/api/demands` 对创意部普通成员做需求类型范围过滤，不只依赖前端隐藏。
- 前端需求列表对创意部普通成员默认锁定创意部，并展示岗位视图提示。
- 后台用户管理已增加“岗位视图”配置，管理员可直接给用户分配：
  - 设计 / UI / 美工 / Banner
  - 视频剪辑
  - 全部创意需求
- 后端和前端均读取 `users.position` 判断岗位视图，不再依赖邮箱映射。
- 创意部负责人和管理员默认可看全部创意需求。

## 4. 后续优化需求

### 4.1 动态字段类型增强

新增字段类型：

- `copy_template`：结构化文案模板，支持产品名、标题、小标题、折扣、卖点、按钮文案、备注等。
- `image_upload`：多图上传，支持数量限制、预览、删除、必填。
- `asset_picker`：从素材库/NAS 选择图片，可作为 `image_upload` 的数据来源。
- `link_or_upload`：共享盘链接或上传文件二选一。
- `file_upload`：通用文件上传。
- `user_select`：选择部门成员，用于设计师/剪辑师等执行人字段。

### 4.2 字段配置增强

字段配置支持：

- 是否必填。
- 是否可筛选。
- 是否在列表展示。
- 文件类型限制。
- 图片数量限制。
- 是否允许素材库选择。
- 是否允许上传。
- 链接规则提示和校验。
- 字段说明文案。

### 4.3 字段组校验

支持跨字段校验：

- 原素材：上传图片或共享盘链接至少填写一个。
- 参考图：至少上传或选择 1 张。
- 文案模板：指定核心项必填。
- 尺寸、设计版式、素材数量同时必填。
- 禁止填写谷歌网盘链接、普通网站链接或压缩包作为原素材。

### 4.4 草稿自动保存

目标：

- 用户填写动态表单时自动保存草稿。
- 关闭页面后可恢复。
- 提交后草稿转正式需求。

实现方向：

- 复用现有 `demand_drafts`。
- 前端定时保存当前表单 payload。
- 草稿列表支持继续编辑、删除、确认提交。

### 4.5 素材库与 NAS

素材库目标：

- 支持品牌、分类、产品型号、文件夹多级浏览。
- 图片上传到当前文件夹。
- 在参考图、产品白底图、原素材字段下方折叠展示。
- 选择素材后写入需求字段或附件引用。

NAS 接入方向：

- 优先调研 NAS 是否支持 WebDAV 或 HTTPS API。
- 若只有 SMB，需要额外内网文件网关，不适合 Cloudflare Edge 直接访问。
- 第一版先做文件夹浏览、图片预览、选择引用，不做完整资产管理。

### 4.6 设计师处理体验

优化项：

- 需求详情增加上一条/下一条。
- 从列表进入详情后，返回时保持筛选、分页和视图。
- 列表快速分配不直接做整行下拉，采用按钮 + 弹层，降低误触和性能压力。
- “整理为 PSD”先作为处理动作：支持生成素材整理包或上传 PSD 链接。

## 5. 实施 Plan

### Step 1：创意部岗位视图分流

状态：已实现，并已改为后台可配置岗位视图。

范围：

- 后端按创意部岗位过滤需求类型。
- 前端默认锁定对应岗位视图。
- 后台用户管理支持维护用户岗位视图。
- 保留管理员/负责人查看全部。

验证：

- `npm run build` 通过。

### Step 2：动态字段高级类型最小闭环

范围：

- 增加 `copy_template`、`image_upload`、`link_or_upload`、`user_select` 的渲染和提交。
- 创意部美工模板切换到高级字段类型。
- 详情页按高级字段类型展示。

### Step 3：字段组校验

范围：

- 扩展字段配置 `validationRules`。
- 新建/编辑/草稿确认统一校验。
- 原素材二选一、参考图必填、文案核心项必填。

### Step 4：自动保存草稿

范围：

- 新建需求页动态表单自动保存。
- 草稿恢复、删除、提交。
- 与现有粘贴草稿共用 `demand_drafts`。

### Step 5：素材库/NAS 最小接入

范围：

- 先确定 NAS 接入协议。
- 实现素材目录浏览、图片上传、选择引用。
- 作为 `asset_picker` 字段组件使用。

### Step 6：处理视图增强

范围：

- 详情上一条/下一条。
- 返回列表状态保持。
- 列表快速分配弹层。
- PSD 整理包或整理动作。

## 6. 风险与约束

- 岗位视图目前服务创意部需求类型分流，后续其他部门如需类似能力，应继续复用用户岗位、需求类型和动态字段配置，而不是新增独立部门系统。
- Cloudflare Edge 无法直接访问内网 SMB NAS，NAS 需要 WebDAV/API 或中间网关。
- 图片和素材能力会增加存储与权限复杂度，需要在 RLS 和文件访问策略上单独设计。
- PSD 自动生成不能承诺完整可编辑分层，产品文案应定位为“PSD 整理板”或“素材整理包”。

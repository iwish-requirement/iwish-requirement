# 前端移动端优先与自适应界面规范\n\n本规范用于约束后续所有新页面和大改版页面的布局与交互，确保系统在手机端与桌面端都具备良好体验，并与当前实现保持一致风格。\n\n## 1. 总体原则\n\n- 移动端优先（Mobile-First）：设计与实现以宽度 < 768px 的手机视口为起点，再通过 `md`、`lg` 断点增强。\n- 不破坏已稳定功能：已有已验证页面（如登录页、当前评分页、用户管理）能不改就不改，确实需要调整时优先只加响应式类或小范围改动。\n- 表格仅在中大屏展示：小屏优先用卡片与横向滚动容器，表格版在 `md:` 及以上显示。\n- 固定导航：顶部导航固定；侧边/底部导航在移动端收纳为抽屉或折叠。\n\n## 2. 布局模式\n\n### 2.1 列表与详情\n\n- 小屏（默认）：\n  - 列表使用纵向卡片布局，每个卡片展示关键字段和操作按钮。\n  - 长字段或多列信息可拆为多行显示，或在卡片内部增加横向滚动区域。\n- 中大屏（`md:` 及以上）：\n  - 列表可使用表格形式展示更多列。\n  - 使用 `overflow-x-auto` 包裹表格，避免列过多导致布局破坏。\n  - 可采用 `hidden md:block` / `md:hidden` 切换卡片和表格版本。\n\n### 2.2 顶部工具条与筛选区域\n\n- 小屏：\n  - 工具条与筛选组件拆分为上下两段，防止一行挤满。\n  - 如遇多个 Tab/筛选项，使用 `overflow-x-auto` + `whitespace-nowrap` 实现横向滑动而非换行。\n- 中大屏：\n  - 横向排列筛选条件和操作按钮，充分利用宽度。\n\n## 3. 导航与交互\n\n- 顶部导航栏固定在页面顶部，包含系统标题、返回/菜单按钮等核心入口。\n- 侧边导航在小屏上以抽屉/折叠菜单形式出现，避免占用过多横向空间。\n- 所有按钮使用 `<button>` 元素，设置足够的点击区域（高度建议 ≥ 40px）。\n- 禁止使用浏览器原生弹窗（`alert/confirm`），统一使用项目内弹窗组件。\n\n## 4. Tailwind 使用约定\n\n- 默认样式为小屏样式，在此基础上添加 `md:`、`lg:` 等断点类。\n- 布局优先使用 `flex`：\n  - 列表/卡片/表格容器使用 `flex` + `flex-col` 或 `flex-row` 组合。\n  - 在中大屏通过 `md:flex-row`、`md:grid` 等实现多列布局。\n- 长内容或多列时使用：\n  - `overflow-x-auto` + `whitespace-nowrap` 实现横向滚动。\n  - 在小屏避免强制换行导致布局难以阅读。\n\n## 5. 按钮与表单交互\n\n- 按钮：\n  - 小屏确保点击区域足够大，左右留出合理 padding。\n  - 禁用态使用明显的视觉弱化（透明度、颜色），并设置 `cursor-not-allowed`。\n- 输入框：\n  - 使用 `<input>` 与 `<textarea>` 元素，避免用 `<div>` 模拟输入框。\n  - 统一去掉默认 outline，通过 Tailwind 自定义 `focus` 状态样式。\n\n## 6. 兼容现有实现\n\n- 对于已实现的用户管理、评分页等，在后续开发中：\n  - 仅在添加新功能或字段时，按本规范调整局部布局。\n  - 不进行大规模重构，避免引入新的视觉或交互问题。
